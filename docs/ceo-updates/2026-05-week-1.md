## Slide 2 — Why this is hard (the moat)

**Headline:**
> Power BI's authoring format is closed. No public spec. No public API. Every shape was reverse-engineered.

**On demand:**
- Microsoft never published the spec for how Power BI stores reports on disk. There is no developer API to author reports programmatically.
- Every visual shape, every field binding, every filter literal, every theme key — figured out empirically against Desktop's behavior.
- We've encoded that empirical knowledge as guardrails the AI cannot violate. That body of knowledge is the moat.
- Byte-fidelity test: we can round-trip any Power BI report file without drift. This isn't a parser — it's an authoring engine.

## Slide 3 — Architecture: two layers, two streams

**Headline:**
> Power BI has two layers. We own one fully, orchestrate the other. Every weekly update splits along this line.

**On demand:**
- **Modeling layer** — data (tables, columns, DAX measures, relationships). Microsoft ships a server we drive. Our value-add: validator stack that doesn't exist anywhere else.
- **Report layer** — visuals (pages, charts, filters, themes, bookmarks). 100% ours; 100% reverse-engineered.
- Every weekly delta from here on is reported as: *Modeling shipped / Modeling next / Report shipped / Report next.*

## Slide 7 — Demo / proof point

**Headline:**
> Same prompt. Microsoft's tools silently produce wrong numbers. Our doctor catches it before Desktop ever sees it.

**On demand — what to show:**
1. **Scenario:** "Show me sales vs target by sub-category."
2. **The hidden trap:** Targets table is tracked at the *Category* level, but the user asked for *Sub-Category*. Math is impossible — the right answer literally cannot be computed.
3. **Microsoft's path:** Power BI BPA, Tabular Editor, Microsoft's MCP — all silent. Dashboard renders. Every row shows the wrong number. Totals don't reconcile.
4. **Our path:** Model-doctor inspects the relationship structure, computes which axes the bridge can cover, returns the list of *blocked axes*. AI refuses to bind the bridged measure on a blocked axis. Dashboard correct on first try.
5. **Show format:** two screenshots side-by-side (broken vs. our output), or a 60-second live demo if the room can handle it. Recommend screenshots — predictable, no live-demo failure risk.
